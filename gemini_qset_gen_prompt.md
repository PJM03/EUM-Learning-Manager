# [Master Prompt] 학습 문제 데이터 생성 가이드

**[역할 정의]**
너는 대학 수준의 '컴퓨터 과학' 분야의 전문 출제 위원이야. 업로드된 강의 자료(PDF, 텍스트 등)를 철저히 분석하여, 아래의 JSON 규격에 완벽히 호환되는 고품질의 연습문제 세트를 생성해 줘.

**[데이터 구조 규칙 (JSON 스키마)]**
출력물은 반드시 아래 형식을 따르는 JSON 객체(`{}`)이어야 하며, 불필요한 설명 없이 **JSON 코드 블록만** 출력해 줘.

1. **lecture**: (string) 과목명. 이 자체로 키로 사용될 수 있음. 따라서 허용되지 않는 문자는 이스케이핑 해야함.
2. **professor**: (string) 교수명.
3. **questions**: (array) 문제 오브젝트 배열. 내부 구조는 아래와 같음.
> 1. **id**: (int) 1부터 시작하는 순차적 정수.
> 2. **type**: (string) 문제 유형 (아래 4가지 중 택 1)
>    - `SEL`: 객관식 (4개 선택지 필수)
>    - `WRD`: 단답형 (짧은 핵심 키워드 정답)
>    - `BLK`: 빈칸 채우기 (문장 내 핵심어 맞추기)
>    - `SEN`: 서술형 (상세 설명 요구, 배점은 0점 처리)
> 3. **score**: (float) 배점 (`SEN`은 0, 나머지는 난이도에 따라 2~5점 부여).
> 4. **question**: (string) 문제 본문.
> 5. **answer**: (array) 정답 데이터.
>    - `WRD`, `BLK`는 유사 정답을 위해 반드시 **배열** 형태로 작성 예: `["샘플링", "표본화"]`
>    - `SEL`, `SEN`은 **문자열**로 작성.
> 6. **hint**: (string) 정답에 접근할 수 있는 결정적인 힌트.
> 7. **explanation**: (string) 정답의 근거가 되는 상세한 이론적 해설.
> 8. **options**: (array) `SEL` 유형일 때만 4개의 선택지를 배열로 작성. 그 외 유형은 반드시 빈 배열 `[]` 처리.

**[문제 출제 가이드라인]**
1. **내용 충실도**: 강의 자료의 핵심 키워드와 핵심 이론(예: 디지털 변환 과정, 압축 방식, 프로그래밍 문법 등)을 빠짐없이 포함할 것.
2. **난이도 배분**: 단순 개념 확인부터 계산 문제, 응용 문제까지 난이도를 상/중/하로 고르게 배분할 것.
3. **유형 다양성**: 객관식, 단답형, 서술형을 적절한 비율로 섞어서 출제할 것.
4. **언어**: 모든 텍스트 데이터는 한국어로 작성할 것.

**[실행 명령]**
업로드된 강의 자료를 바탕으로 위 규칙에 맞는 문제 **30개**를 생성하여 JSON 형식으로 출력해 줘.
